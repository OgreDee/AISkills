# 添加说明文档

为新增的 Skill 或模块自动生成完整 README，并在项目根 README 中添加简述索引。

## 使用方法

```bash
# 指定 skill 名称
/add_readme lua_hot_reload

# 无参数：自动检测当前未建 README 的 skill 目录
/add_readme
```

## 执行流程

### Step 1: 确定目标

- 有参数：直接使用该名称作为目标 skill/模块目录
- 无参数：扫描项目根目录下所有子目录，找到缺少 `README.md` 的 skill 目录
  - 多个时列出让用户选择
  - 全部已有 README 时提示退出

**变量**（后续步骤引用）：
- `SKILL_NAME`: skill 目录名，如 `lua_hot_reload`
- `SKILL_DIR`: skill 完整路径，如 `./lua_hot_reload/`

### Step 2: 收集信息

**必须**先充分阅读目标目录的所有文件以理解其功能：

1. 读取 `{SKILL_DIR}/SKILL.md` — 获取技能的完整描述、工作流、API
2. 读取 `{SKILL_DIR}/scripts/` 下所有文件 — 了解实现细节
3. 读取 `{SKILL_DIR}/install.sh`（如有）— 了解部署方式和依赖
4. 读取项目根 `README.md` — 了解当前索引结构

如果信息不足以撰写完整 README（如缺少 SKILL.md），向用户询问：
- 这个模块的核心功能是什么？
- 有哪些依赖？
- 使用场景是什么？

### Step 3: 生成 Skill README

在 `{SKILL_DIR}/README.md` 中生成文档，**必须包含以下章节**：

```markdown
# {SKILL_NAME} — {一句话标题}

{一段话描述核心功能和价值}

## 工作原理

{流程图或表格，说明主要步骤}

## 目录结构

{树状展示目录内容及各文件说明}

## 快速开始

### 1. 部署
{部署命令和说明}

### 2. 适配（如需要）
{项目特定的适配点}

### 3. 使用
{基本使用方法}

## 依赖

| 依赖 | 说明 |
|------|------|
| ... | ... |

## 核心组件

{每个关键文件的 API 和用法}

## 安全机制（如适用）

{保护措施、限制}

## 许可

内部工具，仅供项目内部使用。
```

**写作要求**：
- 语言：中文
- 风格：简洁、技术性、面向开发者
- 代码示例必须可直接运行
- 依赖表必须完整准确
- 不要编造未在源码中确认的信息

### Step 4: 更新根 README

编辑项目根目录的 `README.md`，在 `## Skills 列表` 的表格中添加一行：

```markdown
| [{SKILL_NAME}](./{SKILL_NAME}/) | {一句话简述，不超过 50 字} |
```

**规则**：
- 按字母顺序或逻辑分组插入
- 简述应能让读者快速理解这个 skill 做什么
- 不要修改表格中已有的其他行
- 如果表格不存在，创建标准格式的表格

### Step 5: 确认输出

完成后向用户报告：
1. 已创建：`{SKILL_DIR}/README.md`（列出主要章节）
2. 已更新：根 `README.md`（显示新增的行）
3. 提示用户检查内容是否需要调整

## 质量检查

生成内容前进行自检：

- [ ] 依赖列表是否完整？（对照 install.sh 和 SKILL.md）
- [ ] 代码示例是否与源码一致？
- [ ] 是否遗漏了重要的 API 或配置项？
- [ ] 根 README 的简述是否准确反映了核心功能？
- [ ] 没有编造源码中不存在的功能？

## 注意事项

- 如果目标目录已有 README.md，先读取后询问用户是覆盖还是补充
- 不要在 README 中重复 SKILL.md 的全部内容，侧重"快速理解"和"快速上手"
- 保持与项目中其他 README 的风格一致
