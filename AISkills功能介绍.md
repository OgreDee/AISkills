## Skill 1：智能断点调试（Lua + C#）

### 它做了什么？

开发者只需用自然语言描述 Bug，AI 自动完成以下全流程：

```
描述 Bug → AI 读代码分析 → 自动设断点 → 注入正在运行的游戏 → 捕获运行时数据 → AI 分析定位根因
```

**完全不需要**：手动加 print、重启游戏、挂 IDE 调试器、反复复现。

### 实际使用示例

```
开发者: /bp 英雄受击后血量没有扣减

AI 自动执行：
  1. 判定 → Lua 层（涉及战斗逻辑）
  2. 读取 TakeDamage 相关代码
  3. 在关键行设置断点（如 hp 计算处）
  4. 注入到正在运行的 Unity 中
  5. 提示开发者触发一次受击
  6. 自动读取捕获的变量：发现 damageReduction 为 nil
  7. 结论：防御力配置表字段拼写错误导致读取为 nil
```

### 两种调试能力对比

| 能力 | Lua 版 (lua_bp_debug) | C# 版 (cs_bp_debug) |
|------|----------------------|---------------------|
| 拦截方式 | Lua debug hook | HarmonyLib 动态注入 |
| 适用场景 | UI、业务逻辑、配表相关 | 引擎底层、框架层、C# 计算 |
| 条件断点 | 支持（变量比较） | 支持（完整 C# 表达式） |
| 表达式求值 | — | 支持（运行时执行任意 C# 代码） |
| 性能影响 | 低（有 FPS 保护） | 极低（仅拦截目标方法） |
| 对游戏影响 | 无需重启 | 无需重启 |

### 安全保障

- 帧率低于 15fps 自动停止（保护游戏不卡死）
- 达到采集上限自动关闭（防止无限占用）
- 仅在 Editor 环境生效（不影响正式包）
- 随时可手动停止

## Skill 2：Lua 热重载

### 痛点

传统修改 Lua 后的验证流程：

```
修改代码 → 退出 Play Mode → 重新进入 → 手动操作到测试现场 → 验证
                         耗时 2-5 分钟
```

### 有了热重载之后

```
修改代码 → AI 自动检查安全性 → 注入到运行中的游戏 → 立即生效
                         耗时 < 3 秒
```

**不退出游戏，改完立刻看效果。**

### 安全检查（4 条规则）

AI 不会盲目热重载，而是先进行安全评估：

| 检查项 | 风险 | AI 决策 |
|--------|------|---------|
| 全局类注册 | 已有实例不会更新 | 拒绝热重载 |
| 全局类定义 | 同上 | 拒绝热重载 |
| 被其他文件依赖 | 需级联更新 | 拒绝热重载 |
| 构造函数变更 | 已有实例不会重建 | 拒绝热重载 |
| 以上均无风险 | 安全 | 执行热重载 |

失败时自动回滚，不会破坏运行状态。

## 技术架构（简版）

```
┌────────────────────────────────────────────────────────────┐
│                    Claude (AI Agent)                         │
│  分析代码、推理 Bug 原因、生成断点、解读运行时数据             │
└───────────────────────────┬────────────────────────────────┘
                            │ MCP 协议（Model Context Protocol）
                            ▼
┌────────────────────────────────────────────────────────────┐
│                Unity Editor (Play Mode)                      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │ lua_debug    │  │ csharp_debug │  │ lua_hot_reload│      │
│  │ MCP Tool     │  │ MCP Tool     │  │ MCP Tool     │      │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘      │
│         │                  │                  │              │
│         ▼                  ▼                  ▼              │
│  Lua DebugHook      HarmonyLib Patch    HotReload Module    │
│  (运行时断点)        (C# 方法拦截)       (模块替换)           │
└────────────────────────────────────────────────────────────┘
```

**关键设计**：AI 通过 MCP 协议直接与 Unity Editor 通信，无需额外插件或 IDE，开发者无感。

## 部署方式

```bash
# 一键部署到目标 Unity 项目
bash install.sh /path/to/unity/project
```

部署后即可在 Claude Code 中通过斜杠命令使用：

| 命令 | 功能 |
|------|------|
| `/bp 描述Bug` | 智能调试入口（自动判断语言层） |
| `/lua_bp_debug` | Lua 断点调试 |
| `/cs_bp_debug` | C# 断点调试 |
| `/lua_hot_reload` | Lua 热重载 |

## 总结

AISkills 的核心价值：

1. **降低调试门槛** — 不需要掌握复杂调试工具，描述问题即可
